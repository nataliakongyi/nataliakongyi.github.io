nky-portfolio | Publishing Guide v1.0 (October 2025)
============================================================
GITHUB PAGES DEPLOYMENT
------------------------------------------------------------
1. Create a new GitHub repository named `nky-portfolio`.
2. Upload all files preserving folder structure.
3. Commit and push changes to the main branch.

4. Go to:
   Settings → Pages → "Build and deployment"
   Choose branch: main → root directory (/)

5. GitHub will generate your site:
   https://yourusername.github.io/nky-portfolio/

6. Wait ~1–3 minutes, then test in browser.

------------------------------------------------------------
UPDATING CONTENT
------------------------------------------------------------
- Edit HTML or CSS locally.
- Test changes by opening index.html in a browser.
- When ready, push updated files to GitHub.
- Pages automatically redeploy on push.

------------------------------------------------------------
BEST PRACTICES
------------------------------------------------------------
- Keep folder names consistent across EN and ES.
- Use relative links (`../shared/css/style.css`) for portability.
- Optimize all images before uploading.
- Test mobile version on Chrome DevTools.
- Back up all `/shared/docs/` files before structural edits.
